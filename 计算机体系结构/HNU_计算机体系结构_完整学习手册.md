# 0. 使用说明

这门课不要按“零散知识点”去学，而要按下面这条主线去学：

**量化分析基础 → ISA / RISC-V → 五级流水线 → 冒险与优化 → 分支预测 → 动态调度 → Cache / AMAT → Cache 一致性**

原因很简单：

- 前面的量化分析告诉你：体系结构优化到底在优化什么；
- ISA 告诉你：指令到底长什么样、字段怎么分、操作数怎么取；
- 五级流水线告诉你：指令在机器里怎么流动；
- 冒险、分支预测、动态调度告诉你：为什么流水线会卡，怎么让它少卡；
- Cache / AMAT / 一致性告诉你：为什么 CPU 不只是算得快，还要“喂得上数据”。

所以这不是 5 门独立的小课，而是一条连续链条。

## 1.1 第一部分：量化设计与分析基础

### 需要掌握的内容

- **1.3 体系结构的定义**
- **1.4 带宽胜过延迟**
- **1.9 量化原理 / 原则、Amdahl 定律、CPU 运行时间公式**

### 必会知识点

#### 1）什么是体系结构

你要区分：

- **ISA / 体系结构层面**：程序员可见的机器模型
- **微体系结构 / 实现层面**：硬件内部怎么把 ISA 实现出来

考试里常见问法：

- “体系结构和实现有什么区别？”
- “某技术属于 ISA 层面还是实现层面？”

#### 2）为什么“带宽胜过延迟”

核心不是一句口号，而是：

- **延迟**：单次操作完成要多久
- **带宽 / 吞吐量**：单位时间能完成多少操作

现代系统更容易通过**并行化、流水线化、多核化、分层缓存**提高吞吐量，
但单次操作的物理延迟更难显著下降。

#### 3）CPU 运行时间公式

最基础公式：

$$
CPU\ Time = Instruction\ Count \times CPI \times Clock\ Cycle\ Time
$$

考虑存储器停顿时，也常写成：

$$
CPU\ Time = (CPU\ Clock\ Cycles + Memory\ Stall\ Cycles) \times Clock\ Cycle\ Time
$$

其中：

$$
CPU\ Clock\ Cycles = IC \times CPI
$$

你必须能回答：

- 哪些技术在降低 **IC**
- 哪些技术在降低 **CPI**
- 哪些技术在缩短 **Clock Cycle Time**
- 哪些技术在减少 **Memory Stall Cycles**

#### 4）Amdahl 定律

核心公式：

$$
Speedup_{overall} = \frac{1}{(1-f) + \frac{f}{S}}
$$

其中：

- $$f$$：被优化部分原来占总执行时间的比例
- $$S$$：该部分的局部加速比

要真正理解的点：

- **局部优化再强，也受未优化部分限制**
- **要优先优化“经常发生”的部分**

#### 5）量化设计的几个核心原则

这门课里你至少要会这三类表达：

- **充分利用并行**
- **利用局部性**
- **重点优化常见情况**

这三个原则后面会反复出现：

- 流水线、超标量、动态调度 —— 充分利用并行
- Cache —— 利用局部性
- 分支预测、Amdahl —— 重点优化常见情况

### 本部分你要达到的标准

- 会写公式
- 会解释公式中每个量的含义
- 会判断某个技术优化了哪个分量
- 会做加速比计算题
- 会把“带宽 / 延迟 / 吞吐量 / 并行 / Amdahl”连起来解释

## 1.2 第二部分：附录 A——ISA 与 RISC-V 基础

### 需要掌握的内容

- ISA
- R-M、R-R 的优劣
- 内存对齐、大小端
- 寻址方式
- RISC-V 32I 的指令集编码、指令功能、操作数

### 必会知识点

#### 1）R-M 与 R-R

- **R-M（Register-Memory）**：寄存器和内存都可直接做算术逻辑操作数
- **R-R（Register-Register / Load-Store）**：算术逻辑操作只作用于寄存器，访存必须通过 load/store

要会比较：

- R-R 为什么更利于流水线化
- R-M 为什么指令功能上更“直接”但实现更复杂
- 为什么 RISC-V 倾向 R-R / Load-Store

#### 2）内存对齐

要理解：

- 为什么多字节数据往往按其大小对齐
- 不对齐访问可能带来的硬件复杂性和性能代价

#### 3）大小端

要会区分：

- **大端**：高位字节放低地址
- **小端**：低位字节放低地址

常见考法：

- 给一个十六进制数，问按某种字节序在内存里如何存放
- 给内存内容，问读出的值是什么

#### 4）寻址方式

至少熟悉：

- 立即数寻址
- 寄存器寻址
- 直接寻址
- 寄存器间接寻址
- 变址 / 基址 + 偏移寻址

你要能把形如：

- `LB R1, 100(R1)`
- `LB R2, (R2)`
- `LB R3, (R3+R4)`
- `LB R4, (1001)`

都翻译成“先算地址，再取内存，再写回寄存器”的过程。

#### 5）RISC-V 32I 编码

这一块要细学，不能只会大概。

你至少要会：

- 指令基本格式：R / I / S / B / U / J
- 各字段作用：opcode / rd / rs1 / rs2 / funct3 / funct7 / imm
- 常见指令的功能和操作数意义
- 立即数是如何拼接和扩展的
- 为什么不同格式会尽量对齐字段位置

### 本部分你要达到的标准

- 给你一条常见 RISC-V 指令，你能说出功能、操作数、字段位置
- 给你一个编码图，你能分析指令类别和字段分布
- 给你寻址表达式，你能手推出地址和结果
- 给你大小端 / 对齐场景，你能直接作答

## 1.3 第三部分：第三章——指令级并行（ILP）

### 需要掌握的内容

- **3.1 数据冒险、结构冒险、控制冒险；及优化技术**
- **3.2 流水线调度、循环展开**

---

### 3.1 冒险与优化

#### 必会三类冒险

##### 1）数据冒险

- RAW：读后写，真相关
- WAR：写后读，反相关
- WAW：写后写，输出相关

要明白：

- 顺序五级流水线最核心的是 RAW
- WAR / WAW 常在乱序或更复杂流水线里更重要
- 寄存器重命名主要解决的是 **WAR / WAW**

##### 2）结构冒险

本质：

- 多条指令在同一周期争用同一硬件资源

##### 3）控制冒险

本质：

- 分支 / 跳转改变 PC，导致后续已取指令可能无效

#### 优化技术

- forwarding / bypassing
- stall / bubble
- 编译调度
- 循环展开
- 延迟分支
- 静态分支预测
- 动态分支预测
- 动态调度

你要做到：

- 能判断某种优化主要减少哪一类停顿
- 能把某技术放进流水线 CPI 公式里解释

---

### 3.2 流水线调度、循环展开

#### 你要理解的核心

流水线调度不是“改程序功能”，而是：

- 在**不改变语义**的前提下
- 重新安排指令顺序
- 尽量减少停顿

循环展开则是：

- 减少分支次数
- 暴露更多并行性
- 给编译器更多调度空间

#### 你要会的题型

- 给一段循环，做展开
- 调整指令顺序减少 RAW 停顿
- 解释展开为什么能提高性能，也可能增加代码尺寸

---

### 3.3 分支预测

#### 你至少要会

- 为什么需要分支预测
- 静态预测和动态预测的基本区别
- 分支预测错误会带来什么代价
- 分支预测减少的是哪个 CPI 分量

#### 常见考法

- 问“分支预测能不能彻底解决控制冒险”
- 给流水线图，让你分析预测正确 / 错误时冲刷了哪些阶段

### 3.4 / 3.5 动态调度：记分牌与 Tomasulo

这是本课最容易“看懂图但不会做题”的地方。

#### 记分牌算法要掌握什么

- 它是**集中式控制**
- 核心是根据状态表判断：
  - 是否可以发射
  - 是否可以读操作数
  - 是否可以执行
  - 是否可以写回
- 重点关注三类风险：
  - 结构冲突
  - WAW
  - WAR

#### Tomasulo 算法要掌握什么

- 它是**分布式控制**
- 引入**保留站**
- 通过**寄存器重命名**消除 WAR / WAW
- 通过等待源操作数就绪解决 RAW
- 利用 CDB 广播结果

#### 你真正要会的，不是概念，而是“表格推进”

你必须会：

- 写保留站状态
- 写寄存器结果状态
- 写指令状态表
- 给出某周期系统状态变化
- 比较记分牌和 Tomasulo 的异同

### 本部分你要达到的标准

- 给一串指令，能找出冒险
- 能画五级流水线时序图
- 能说明 forwarding 是否足够
- 能对分支预测正确 / 错误做分析
- 能手推记分牌和 Tomasulo 的状态表

## 1.4 第四部分：附录 C——五级流水线与 RISC-V 基本实现

### 需要掌握的内容

- **C.1 五级流水线，流水线 CPI（四个分量）**
- **C.2 流水线冒险：停顿，数据冒险，分支预测**
- **C.3 RISC-V 的基本实现（实验一）**

### 必会知识点

#### 1）五级流水线的 5 个阶段

- IF：取指
- ID：译码 / 读寄存器
- EX：执行 / 地址计算
- MEM：访存
- WB：写回

#### 2）流水线 CPI 分解

$$
Pipeline\ CPI = Ideal\ CPI + Structural\ Stall + Data\ Stall + Control\ Stall
$$

这条式子非常重要，因为你后面几乎所有优化题都能落到它上面。

#### 3）RISC-V 基本实现

你要会结合数据通路图说：

- 各部件干什么
- 一条指令每个阶段取什么值
- 控制信号在哪一级起作用
- hazard 单元什么时候 stall、什么时候 flush

### 本部分你要达到的标准

- 会背 5 级流水线各阶段职责
- 会根据指令类型分析数据通路
- 会把流水线和 RISC-V 指令真正对应起来

## 1.5 第五部分：附录 B——存储器性能与 Cache 优化

### 需要掌握的内容

- **B.2 CPU 执行时间（存储器停顿周期），存储器平均访问时间**
- **B.3 6 种优化缓存的方法**

### 必会知识点

#### 1）平均内存访问时间 AMAT

$$
AMAT = Hit\ Time + Miss\ Rate \times Miss\ Penalty
$$

#### 2）三类缺失

- 冷不命中 / compulsory miss
- 容量不命中 / capacity miss
- 冲突不命中 / conflict miss

#### 3）6 种缓存优化方法

至少会从下面角度理解：

- 是降低命中时间、缺失率，还是缺失代价
- 优点是什么
- 代价是什么

常见包括：

- 增大块大小
- 增大缓存
- 提高相联度
- 多级缓存
- 读缺失优先于写缺失
- 索引缓存时避免地址转换

#### 4）存储器停顿周期

你要会把它接回 CPU 时间公式和 CPI 公式。

### 本部分你要达到的标准

- 会做 AMAT 计算
- 会分析无 Cache / 有 Cache 的平均 CPI
- 会算加速比
- 会做 tag/index/offset 位数题
- 会做命中 / 不命中 / 替换题

## 1.6 第六部分：5.4 Cache 一致性

### 需要掌握的内容

- Cache 一致性的基本问题
- 监听法
- 目录法

### 必会知识点

#### 1）为什么会有一致性问题

多个处理器 / 核心各自持有某个共享数据的缓存副本时：

- 某一方写了数据
- 其他副本可能还是旧值

这就破坏了一致性。

#### 2）监听法

特点：

- 基于总线广播
- 各缓存监听其他处理器对共享块的读写
- 可扩展性一般

#### 3）目录法

特点：

- 通过目录维护共享块的状态和拥有者信息
- 有额外存储开销
- 更适合大规模系统

#### 4）状态转换

你要会根据访问序列手推：

- Invalid / Shared / Exclusive（或 Modified）之间怎么切换
- 是否需要失效他人副本
- 是否需要写回
- 是否发生替换

### 本部分你要达到的标准

- 能解释监听法和目录法各自思路
- 能手推状态变化表
- 能比较它们的优缺点

# 3. 实验题目

目前核心实验资料里，与你本学期主范围最相关的是 **3 个实验**。

---

## 3.1 实验1：RISC-V 指令理解 / RISC-V 流水线

### 实验目标

- 理解 RV32I Core 设计图
- 理解每条指令的数据流和控制信号
- 为后续流水线和乱序发射实验打基础

### 实验内容

#### 1）安装模拟器 Ripes

- 下载
- 配置编译器

#### 2）汇编代码分析

- 看 C 程序
- 生成汇编代码
- 分析 `main` 函数
- 特别分析：
  - 栈空间分配
  - 第一个循环
  - 第二个循环
  - 退出 main

#### 3）各部件及主要控制信号

至少涉及：

- HazardUnit
- ControlUnit
- NPC_Generator
- RegisterFile
- ImmOperandUnit
- BranchDecision
- ALU
- DataExt

#### 4）思考问题

重点分析具体指令在五级流水线中的过程，例如：

- `add x15, x14, x15`
- 某条 `sw` 指令在 IF / ID / EX / MEM / WB 的数据通路与控制信号
- BranchE 信号的作用
- 不同跳转 target 选择的优先级

#### 5）附加思考题

- 哪些冲突需要插入气泡
- stall 和 flush 如何控制
- 静态不跳转预测下 branch 如何处理

### 这个实验真正训练什么

- 把“RISC-V 指令”与“流水线电路”连起来
- 不再只是背指令，而是真正理解一条指令怎么流经机器
- 为附录 C.3 和后续冒险分析打地基

### 你做实验时要重点看什么

- 一条 load/store/add/branch 指令在 5 个阶段分别做什么
- 哪些信号在 EX 起作用，哪些在 MEM / WB 起作用
- forwarding / stall / flush 是怎么触发的

---

## 3.2 实验2：CPU 动态指令调度 Tomasulo

### 实验目标

- 熟悉 Tomasulo 模拟器
- 理解指令流出、执行、写结果三个关键阶段
- 理解 load/store 与浮点指令如何调度
- 能写保留站、指令状态表、浮点寄存器状态表

### 实验给定的典型指令流

```text
L.D   F6, 21(R2)
L.D   F2, 0(R3)
MUL.D F0, F2, F4
SUB.D F8, F6, F2
DIV.D F10, F0, F6
ADD.D F6, F8, F2
```

典型延迟设定：

- 加减：2 周期
- 乘：10 周期
- load/store：2 周期
- 除：40 周期

### 实验要求回答的典型问题

- 第 2、3 周期时，load 部件做了什么改动
- `MUL.D` 刚开始执行时，系统状态和上一周期相比发生了什么变化
- 为什么 `MUL.D` 发射后没有立刻执行
- 第 15、16 周期系统发生了什么变化
- 所有指令什么时候执行结束

### 思考题

- Tomasulo 和 Scoreboard 有什么异同
- Tomasulo 如何解决结构冒险、RAW、WAR、WAW

### 这个实验真正训练什么

- 不只是“知道 Tomasulo 是什么”
- 而是要会逐周期推进
- 你必须能看懂：
  - 哪条指令为什么卡住
  - 谁在等谁
  - 谁的结果通过什么方式广播

### 你做实验时要重点看什么

- 保留站里 `Qj / Qk / Vj / Vk` 的变化
- 寄存器结果状态怎么变化
- 为什么有的指令能先执行，有的不能
- 为什么 Tomasulo 比记分牌更能消除假相关

---

## 3.3 实验3：多 Cache 一致性算法

### 实验目标

- 熟悉监听法和目录法模拟器
- 理解两种一致性机制的基本思想
- 给定读写序列时，能模拟状态变化、写回、替换

### 实验内容

#### 1）预备知识

- MSI
- MESI
- 监听法
- 目录法

#### 2）监听法模拟

给定一串访问序列，要求填写：

- 是否发生替换
- 是否发生写回
- 状态如何变化
- 具体发生了什么事件

典型序列包括：

- CPU A 读第 5 块
- CPU B 读第 5 块
- CPU C 读第 5 块
- CPU B 写第 5 块
- CPU D 读第 5 块
- CPU B 写第 21 块
- CPU A 写第 23 块

#### 3）目录法模拟

给定另一组访问序列，要求填写：

- 目录状态变化
- Cache 块状态变化
- 具体通信过程

典型序列包括：

- CPU A 读第 6 块
- CPU B 读第 6 块
- CPU D 读第 6 块
- CPU B 写第 6 块
- CPU C 读第 6 块

#### 4）思考题

- 目录法和监听法分别有什么优劣

### 这个实验真正训练什么

- 会根据访问序列**手推状态机**
- 会理解**为什么要失效、为什么要写回、为什么目录要更新**
- 会比较监听法和目录法在通信方式与可扩展性上的区别

### 你做实验时要重点看什么

- 读命中 / 读不命中 / 写命中 / 写不命中时状态如何变化
- 什么时候需要让别人失效
- 什么时候需要把最新副本写回
- 目录里记录的是谁持有共享 / 独占
