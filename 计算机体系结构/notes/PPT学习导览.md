## 第一章：量化计算与分析基础

### 你要抓住的主线

计算机体系结构不是单纯背硬件名词，而是在约束下做设计权衡：性能、功耗、成本、可靠性、兼容性。课件第一章的核心是：一切架构设计都要能被量化比较。

### 核心概念

- 架构/ISA：程序员可见的软硬件接口，例如寄存器、寻址方式、指令格式、控制流指令。
- 微架构：ISA 的具体硬件实现，例如流水线、缓存、总线、执行顺序、调度、功耗控制。
- 计算机分类：个人移动设备、桌面计算机、服务器、仓库级计算机/集群、超级计算机、嵌入式设备。
- 并行性：ILP 指令级并行，DLP 数据级并行，TLP 线程级并行，RLP 请求级并行。
- Flynn 分类：SISD、SIMD、MISD、MIMD。

### 必背公式

```text
CPU Time = Instruction Count * CPI * Clock Cycle Time
CPU Time = Instruction Count * CPI / Clock Rate

Speedup = Execution Time_old / Execution Time_new

Amdahl Speedup = 1 / ((1 - F) + F / S)

MTBF = MTTF + MTTR
Availability = MTTF / MTBF

Dynamic Energy = 1/2 * C * V^2
Dynamic Power  = 1/2 * C * V^2 * f
```

### 学习重点

- 响应时间和吞吐率不是一回事。跑车响应时间好，校车吞吐率可能更高。
- MIPS 不适合作为跨机器性能比较的唯一指标，因为它依赖 ISA、程序和 CPI。
- benchmark 越接近真实程序越可靠；SPEC 常用几何平均来综合性能比。
- Amdahl 定律说明：只优化小比例部分，整体收益会很快碰到上限。
- 功耗不只是频率问题，电压平方项非常关键；降频通常降低功率，不一定降低每次任务的能量。

## 附录 A：ISA 与 RISC-V

### 你要抓住的主线

程序的本质可以粗略看成“指令 + 数据”。ISA 决定指令如何表达、数据如何访问、控制流如何跳转，也决定编译器和硬件之间如何配合。

### ISA 设计维度

- 操作数存放位置：栈型、累加器型、通用寄存器型。
- 通用寄存器型分类：Register-Memory 与 Load-Store。
- 存储器寻址：字节编址、端序、对齐。
- 寻址方式：立即数、偏移寻址、寄存器间接寻址、PC 相对寻址等。
- 操作数类型与大小：整数、浮点、字符、向量、BCD。
- 操作类型：算术、逻辑、数据传输、控制流、系统/CSR 等。
- 指令编码：定长、变长、混合编码。
- 编译器关系：寄存器分配、变量寻址、控制结构频率会反过来影响 ISA 设计。

### RISC-V 重点

- RISC-V 是 Load-Store 架构：普通 ALU 指令只操作寄存器，只有 Load/Store 访问内存。
- 基本指令格式：R、I、S、B、U、J。
- 常见字段：`opcode`、`rd`、`rs1`、`rs2`、`funct3`、`funct7`、`imm`。
- R-type：寄存器到寄存器运算，例如 `add x9, x20, x21`。
- I-type：立即数运算、load、`jalr`、CSR。
- S-type：store，立即数被拆成两段。
- B-type：条件分支，分支偏移的立即数字段不是连续摆放。
- U-type：高 20 位立即数，例如 `lui`。
- J-type：无条件跳转，例如 `jal`。

### 函数调用与 ABI

- 参数优先放在 `a0-a7`。
- 返回值通常放在 `a0-a1`。
- `jal` 用于跳转并保存返回地址。
- `jalr x0, 0(x1)` 常用于函数返回。
- callee 需要保存并恢复保存寄存器；caller 如果之后还要用临时寄存器，需要自己保存。

## 流水线基础

### 你要抓住的主线

流水线的目标是提高吞吐率，不是缩短单条指令的延迟。五级流水线把指令拆成 IF、ID、EX、MEM、WB，让多条指令在不同阶段重叠执行。

### 五级阶段

- IF：取指，PC 通常加 4。
- ID：译码，读寄存器，部分设计会尽早做分支判断。
- EX：ALU 运算或有效地址计算。
- MEM：Load/Store 访存。
- WB：把 ALU 或 Load 结果写回寄存器。

### 流水线寄存器

- `IF/ID`
- `ID/EX`
- `EX/MEM`
- `MEM/WB`

它们保存阶段间传递的数据和控制信号。例如目的寄存器 `rd` 在 ID 阶段就知道，但到 WB 阶段才用到，所以必须一路跟着指令往后传。

### 性能直觉

```text
理想 k 级流水线完成 n 个任务约需 k + n - 1 个周期
理想最大加速比约等于流水线深度 k
实际加速比会被阶段不均衡、流水线寄存器开销、冒险和异常拉低
```

课件中的关键反直觉点：单周期一条指令可能 12ns，五级流水线一条指令可能 15ns，但程序整体更快，因为吞吐率提升了。

## 流水线冒险

### 三类冒险

| 类型 | 含义 | 常见处理 |
| --- | --- | --- |
| 结构冒险 | 同一周期争用同一硬件资源 | 增加硬件资源，例如分离指令/数据存储器 |
| 数据冒险 | 指令间存在数据依赖 | 转发、停顿、编译器调度 |
| 控制冒险 | 下一条指令取决于分支结果 | 尽早判断、分支预测、延迟分支、flush |

### 数据冒险重点

- RAW：写后读，最常见，也最直观。
- WAR：读后写，在简单顺序五级流水线中一般不出现，但乱序执行中会出现。
- WAW：写后写，在乱序完成中会出现。
- Forwarding 可以把 EX/MEM 或 MEM/WB 的结果提前送给后续 EX 阶段。
- Load-use 冒险常常无法完全靠转发解决，因为数据在需要时还没从内存回来，不能“向过去转发”。

### 控制冒险重点

- 预测不跳转：如果正确，几乎无代价；如果错误，需要 flush。
- 动态预测会记录分支历史，常见思想是 2 位饱和计数器。
- 延迟分支把一条安全指令填入延迟槽，减少分支惩罚。

### 异常重点

精确异常要求：异常指令之前的指令都像已经完成，异常指令之后的指令都像没有执行。流水线、延迟分支、乱序执行都会让这个目标变难。

## ILP：记分牌与 Tomasulo

### ILP 的基本公式

```text
Pipeline CPI =
Ideal Pipeline CPI
+ Struct Stalls
+ RAW Stalls
+ WAR Stalls
+ WAW Stalls
+ Control Stalls
+ Memory Stalls
```

研究 ILP 的核心问题是：如何减少各种 stall，让无关指令重叠执行。

### 记分牌 Scoreboard

记分牌是集中式控制，典型特征是顺序发射、乱序执行、乱序完成。

四个阶段：

1. Issue：译码，检测结构冒险和 WAW。
2. Read operands：等源操作数可用，解决 RAW。
3. Execution：功能部件执行。
4. Write result：写回前检测 WAR。

三类状态表：

- Instruction status：每条正在执行的指令处于哪个阶段。
- Functional unit status：功能部件是否忙、操作类型、目的寄存器、源寄存器、源操作数来自哪里、是否就绪。
- Register result status：哪个功能部件将写某个寄存器。

### Tomasulo

Tomasulo 是分布式控制，核心组件是保留站、寄存器重命名、公共数据总线 CDB。

三个阶段：

1. Issue：找到空闲保留站，发射指令，并把寄存器名替换成值或生产者标签。
2. Execute：当 `Qj = 0` 且 `Qk = 0`，说明操作数就绪，可以执行。
3. Write result：执行结果通过 CDB 广播给所有等待者，同时更新寄存器结果状态。

为什么它强：

- 用寄存器重命名消除 WAR 和 WAW。
- 用保留站等待操作数，缓解 RAW。
- 结果广播给所有等待者，比单纯写寄存器更灵活。

为什么它也有代价：

- 硬件复杂。
- CDB 可能成为性能瓶颈。
- 要支持精确异常，还需要额外机制，例如重排序缓冲思想。

### Scoreboard 与 Tomasulo 对比

| 维度 | Scoreboard | Tomasulo |
| --- | --- | --- |
| 控制方式 | 集中式 | 分布式 |
| 发射 | 顺序发射 | 通常仍按序发射到保留站 |
| 执行/完成 | 乱序执行、乱序完成 | 乱序执行、乱序完成 |
| WAR/WAW | 通过停顿避免 | 通过寄存器重命名避免 |
| RAW | 等操作数可用 | 保留站监听 CDB |
| 结果传递 | 主要围绕寄存器/功能部件状态 | CDB 广播 |

## 自测题

1. ISA 和微架构有什么区别？请用 RISC-V 与流水线各举一个例子。
2. 为什么响应时间更短不一定代表吞吐率更高？
3. `CPU Time = IC * CPI / Clock Rate` 中，编译器、ISA、组织结构、工艺分别可能影响哪些项？
4. Amdahl 定律中，如果可优化部分比例很小，为什么局部加速再大也没用？
5. 为什么 MIPS 可能与真实性能相反？
6. RISC-V 为什么只有 Load/Store 指令访问内存？
7. R、I、S、B、U、J 六种格式各自主要服务什么指令？
8. `A[12] = h + A[8]` 这类代码为什么数组偏移要乘元素大小？
9. 流水线为什么提高吞吐率而不是单条指令执行速度？
10. 为什么需要 `IF/ID`、`ID/EX`、`EX/MEM`、`MEM/WB` 这些流水线寄存器？
11. 为什么 Load-use 冒险无法总靠转发解决？
12. 分支预测错误后为什么要 flush？
13. 精确异常的“精确”到底精确在哪里？
14. 记分牌四阶段分别检测或处理哪些冒险？
15. Tomasulo 如何用保留站和 CDB 让多个等待者同时获得结果？

## 复习建议

第一轮只看概念和公式，不纠结每张图的所有信号名。第二轮重点手推例题：Amdahl、CPI、RISC-V 指令翻译、流水线周期图、冒险判断。第三轮专攻记分牌和 Tomasulo 的 cycle-by-cycle 示例，因为这部分最容易“看懂了但不会做题”。

如果时间紧，优先顺序是：性能公式与 Amdahl、ISA/RISC-V 指令格式、五级流水线与冒险、Scoreboard 四阶段、Tomasulo 三阶段。
